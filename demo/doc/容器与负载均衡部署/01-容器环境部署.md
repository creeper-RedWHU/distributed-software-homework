# 容器环境部署

## 零、macOS 安装 Docker 环境

### 方式一：Docker Desktop（GUI，适合新手）

```bash
# Homebrew 安装
brew install --cask docker
```

安装后从启动台打开 Docker Desktop，等待右上角鲸鱼图标变为稳定状态。

### 方式二：Colima（轻量命令行，推荐开发者）

```bash
# 安装 Colima + Docker CLI + docker-compose
brew install colima docker docker-compose

# 启动 Colima（分配 4核 CPU、4GB 内存、60GB 磁盘）
colima start --cpu 4 --memory 4 --disk 60
```

### 验证安装

```bash
# Docker 版本
docker --version
# Docker version 29.x.x

# Compose 版本（Docker Desktop 用 docker compose，Colima 用 docker-compose）
docker compose version   # Docker Desktop
docker-compose version   # Colima

# Colima 用户检查运行状态
colima status
```

> **注意**：后续命令中，Docker Desktop 用户使用 `docker compose`（空格），Colima 用户使用 `docker-compose`（连字符）。

## 三、Dockerfile 说明

项目使用**多阶段构建**优化镜像大小：

```dockerfile
# 阶段1: 使用Maven构建项目（非alpine镜像，兼容ARM64/Apple Silicon）
FROM maven:3.9-eclipse-temurin-17 AS builder
WORKDIR /build
COPY pom.xml .
RUN mvn dependency:go-offline -B       # 先下载依赖（利用Docker缓存层）
COPY src ./src
RUN mvn clean package -DskipTests -B

# 阶段2: 运行阶段，仅包含JRE
FROM eclipse-temurin:17-jre
WORKDIR /app
COPY --from=builder /build/target/*.jar app.jar
# 支持通过 SERVER_PORT 环境变量配置端口
ENV SERVER_PORT=8080
ENTRYPOINT ["sh", "-c", "java $JAVA_OPTS -Dserver.port=${SERVER_PORT} -jar /app/app.jar"]
```

关键设计点：
- **多阶段构建**：最终镜像不包含Maven和源代码，体积更小
- **依赖层缓存**：先复制 `pom.xml` 下载依赖，利用Docker层缓存加速后续构建
- **端口可配置**：通过 `SERVER_PORT` 环境变量在运行时指定端口

## 四、docker-compose.lb.yml 说明

### 4.1 服务组成

| 服务 | 镜像 | 容器端口 | 宿主机端口 | 说明 |
|------|------|---------|-----------|------|
| mysql | mysql:8.0 | 3306 | 3306 | MySQL数据库 |
| redis | redis:7-alpine | 6379 | 6379 | Redis缓存 |
| app1 | 本地构建 | 8081 | 8081 | 后端实例1 |
| app2 | 本地构建 | 8082 | 8082 | 后端实例2 |
| nginx | nginx:alpine | 80 | 80 | Nginx反向代理 |

### 4.2 服务依赖关系

```
mysql(healthy) ──┐
                 ├──► app1(healthy) ──┐
redis(healthy) ──┤                    ├──► nginx
                 ├──► app2(healthy) ──┘
                 └────────────────────
```

使用 `depends_on` + `condition: service_healthy` 确保启动顺序正确。

### 4.3 一键启动

```bash
cd demo

# 构建并启动所有容器
docker-compose -f docker-compose.lb.yml up -d --build

# 查看容器状态
docker-compose -f docker-compose.lb.yml ps

# 查看日志
docker-compose -f docker-compose.lb.yml logs -f

# 查看某个服务日志
docker-compose -f docker-compose.lb.yml logs -f app1
docker-compose -f docker-compose.lb.yml logs -f app2

# 停止所有容器
docker-compose -f docker-compose.lb.yml down

# 停止并清除数据卷
docker-compose -f docker-compose.lb.yml down -v
```

### 4.4 验证部署

```bash
# 通过Nginx访问（80端口）
curl http://localhost/api/product/list

# 直接访问后端实例
curl http://localhost:8081/api/product/list
curl http://localhost:8082/api/product/list

# 查看响应头中的X-Server-Port确认请求分发
curl -i http://localhost/api/product/server-info

# 访问前端页面
open http://localhost
```

## 五、环境变量配置

后端服务通过以下环境变量配置连接信息：

| 环境变量 | 说明 | 示例 |
|---------|------|------|
| SERVER_PORT | 后端服务端口 | 8081 |
| SPRING_DATASOURCE_URL | 数据库连接URL | jdbc:mysql://mysql:3306/seckill_db... |
| SPRING_DATASOURCE_USERNAME | 数据库用户名 | root |
| SPRING_DATASOURCE_PASSWORD | 数据库密码 | root |
| SPRING_REDIS_HOST | Redis主机地址 | redis |
| SPRING_REDIS_PORT | Redis端口 | 6379 |

## 六、常见问题

### Q1: 容器启动后App无法连接MySQL？
A: MySQL初始化需要时间。docker-compose中已配置健康检查和依赖等待，通常需要等30-60秒。可通过 `docker-compose -f docker-compose.lb.yml logs -f mysql` 查看MySQL是否就绪。

### Q2: 端口冲突？
A: 确保宿主机的 80、3306、6379、8081、8082 端口未被占用：
```bash
lsof -i :80
lsof -i :3306
```

### Q3: 镜像构建失败？
A: 确保Maven依赖能正常下载。如果网络问题，可在 `pom.xml` 中配置国内镜像源。
