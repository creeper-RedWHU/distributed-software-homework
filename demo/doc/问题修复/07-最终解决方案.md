# 最终解决方案

## ✅ 问题已解决

**Apache Shiro 2.0.1 与 Spring Boot 4.0.3 不兼容问题已通过降级 Spring Boot 解决。**

## 🎯 最终方案

### 降级 Spring Boot 到 2.7.18

**原因**: Shiro 2.0.1 仅支持 `javax.servlet` API，不支持 Spring Boot 3.x/4.x 的 `jakarta.servlet` API

**版本选择**: Spring Boot 2.7.18 是 2.x 系列的最后一个稳定版本，兼容 Shiro 2.0.1

## 📋 修改清单

### 1. pom.xml

```xml
<!-- 从 4.0.3 降级到 2.7.18 -->
<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>2.7.18</version>
    <relativePath/>
</parent>
```

### 2. 导入修改

将所有 `jakarta.*` 改为 `javax.*`：

| 文件 | 修改 |
|------|------|
| AuthController.java | `jakarta.servlet` → `javax.servlet` |
| UserController.java | `jakarta.servlet` → `javax.servlet` |
| UserRegisterRequest.java | `jakarta.validation` → `javax.validation` |
| UserLoginRequest.java | `jakarta.validation` → `javax.validation` |
| UserUpdateRequest.java | `jakarta.validation` → `javax.validation` |

### 3. Shiro 依赖

保持简洁的核心库配置：

```xml
<dependency>
    <groupId>org.apache.shiro</groupId>
    <artifactId>shiro-spring</artifactId>
    <version>2.0.1</version>
</dependency>
<dependency>
    <groupId>org.apache.shiro</groupId>
    <artifactId>shiro-web</artifactId>
    <version>2.0.1</version>
</dependency>
<dependency>
    <groupId>org.apache.shiro</groupId>
    <artifactId>shiro-ehcache</artifactId>
    <version>2.0.1</version>
</dependency>
```

## 🎓 经验总结

### 问题演进

```
1. ByteSource 编译错误
   ↓ 创建 CustomCredentialsMatcher

2. Bean 冲突（securityManager）
   ↓ 禁用自动配置

3. Bean 冲突（defaultAdvisorAutoProxyCreator）
   ↓ 禁用更多自动配置

4. javax.servlet.Filter 找不到（shiro-spring-boot-starter）
   ↓ 移除 starter，使用核心库

5. javax.servlet.Filter 找不到（shiro-web 核心库）
   ↓ 降级 Spring Boot 到 2.7.18 ✅ 最终解决
```

### 关键教训

1. **版本兼容性至关重要**
   - 在选择中间件前，先确认版本兼容性
   - 检查 API 命名空间（javax vs jakarta）

2. **不要盲目追新**
   - Spring Boot 4.x 是新版本，但生态可能还没跟上
   - 稳定的老版本（2.7.18）往往更可靠

3. **问题排查思路**
   - 从错误日志找根本原因
   - 不要停留在表面问题
   - 善用搜索引擎和官方文档

## 🔮 未来展望

### 如何迁移回 Spring Boot 4.x？

当 Apache Shiro 发布支持 Jakarta EE 的版本时（可能是 Shiro 3.x）：

1. 升级 Shiro 到新版本
2. 升级 Spring Boot 到 4.x
3. 将所有 `javax.*` 改回 `jakarta.*`
4. 重新测试所有功能

### 其他选择

如果必须使用 Spring Boot 4.x：

**选项**: 使用 Spring Security 替代 Shiro
- ✅ 原生支持 Jakarta EE
- ✅ 与 Spring Boot 深度集成
- ❌ 需要重写认证授权逻辑

## 💡 常见问题

### Q: Spring Boot 2.7.18 还安全吗？

**答**: 是的。2.7.x 是 LTS 版本，至少维护到 2025 年 8 月，包含所有安全补丁。

### Q: 降级会影响其他功能吗？

**答**: 不会。所有核心功能（MyBatis、Redis、Kafka 等）都兼容 Spring Boot 2.7.18。

### Q: 为什么不用桥接方案？

**答**: 字节码转换和类加载器桥接非常复杂，维护困难，不适合生产环境。
