## 📋 配置检查

### pom.xml 依赖配置

**重要**: 使用 Shiro 核心库，**不使用** `shiro-spring-boot-starter`

```xml
<!-- Apache Shiro (不使用 starter) -->
<dependency>
    <groupId>org.apache.shiro</groupId>
    <artifactId>shiro-spring</artifactId>
    <version>2.0.1</version>
</dependency>
<dependency>
    <groupId>org.apache.shiro</groupId>
    <artifactId>shiro-web</artifactId>
    <version>2.0.1</version>
</dependency>
<dependency>
    <groupId>org.apache.shiro</groupId>
    <artifactId>shiro-ehcache</artifactId>
    <version>2.0.1</version>
</dependency>
```

**原因**: `shiro-spring-boot-starter` 与 Spring Boot 4.0.3（Jakarta EE）存在兼容性问题

## ⚠️ 常见问题

### Q1: 启动报 javax.servlet.Filter 找不到

**原因**: 使用了 `shiro-spring-boot-starter`

**解决**:
1. **检查 pom.xml**，确保使用的是核心库而不是 starter
   ```bash
   grep -A 3 "shiro-spring" pom.xml
   ```

   应该看到 `shiro-spring`、`shiro-web`、`shiro-ehcache`，而不是 `shiro-spring-boot-starter`

2. **更新依赖**
   ```bash
   mvn clean
   mvn dependency:resolve
   ```

3. **重启 IDE**
   - IntelliJ IDEA: File > Invalidate Caches > Restart

### Q2: 密码验证失败

**检查**：
1. 数据库中的测试数据是否正确初始化
2. 运行密码生成脚本验证加密：
   ```bash
   ./generate_password.sh
   ```

### Q3: 权限注解不生效

**检查**：
1. `ShiroConfig` 中有 `authorizationAttributeSourceAdvisor` Bean
2. `ShiroConfig` 中有 `defaultAdvisorAutoProxyCreator` Bean
3. 检查日志是否有 AOP 相关错误

### Q4: 数据库连接失败

**检查**：
1. MySQL 是否运行：`mysql -u root -p`
2. 数据库密码是否正确：`application.yml` 中的 `password`
3. 数据库 `distributed` 是否存在（会自动创建）
