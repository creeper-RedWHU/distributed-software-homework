# Apache Shiro 登录认证系统

本项目使用 Apache Shiro 实现用户登录认证和权限管理功能。

## 系统架构

### 数据库表设计

#### 1. 用户信息表（t_user）
存储用户的基本信息，如昵称、手机、邮箱等。

```sql
CREATE TABLE t_user (
    id             BIGINT PRIMARY KEY,
    username       VARCHAR(100) UNIQUE,
    password_hash  VARCHAR(256),
    nickname       VARCHAR(100),
    phone          VARCHAR(20),
    email          VARCHAR(200),
    status         TINYINT,
    ...
);
```

#### 2. 用户登录表（t_user_login）
存储用户的登录凭证和类型信息。

```sql
CREATE TABLE t_user_login (
    id          BIGINT PRIMARY KEY,
    user_id     BIGINT,
    username    VARCHAR(100) UNIQUE,
    password    VARCHAR(256),
    user_type   VARCHAR(20),  -- MERCHANT/BUYER/ADMIN
    salt        VARCHAR(64),
    status      TINYINT,
    ...
);
```

### 用户类型与权限

| 用户类型 | 代码 | 角色 | 权限 |
|---------|------|------|------|
| 商家 | MERCHANT | MERCHANT | product:create, product:update, product:delete, product:view, order:view, order:process |
| 买家 | BUYER | BUYER | product:view, order:create, order:view, order:cancel |
| 管理员 | ADMIN | ADMIN | 所有权限 (user:*, product:*, order:*, system:*) |

## Shiro 核心组件

### 1. UserRealm（认证和授权）
- **认证（Authentication）**：验证用户登录，检查用户名和密码
- **授权（Authorization）**：获取用户的角色和权限信息

位置：`com.example.demo.shiro.UserRealm`

### 2. ShiroConfig（配置）
配置 Shiro 的 SecurityManager、过滤器链、密码加密等。

位置：`com.example.demo.config.ShiroConfig`

### 3. AuthService（业务逻辑）
处理注册、登录、登出、修改密码等业务逻辑。

位置：`com.example.demo.service.AuthService`

## 使用注解进行权限控制

### 1. 需要认证（登录）
```java
@RequiresAuthentication
public Result<String> someMethod() {
    // 需要登录才能访问
}
```

### 2. 需要特定角色
```java
@RequiresRoles("MERCHANT")
public Result<String> merchantMethod() {
    // 需要商家角色
}

@RequiresRoles(value = {"MERCHANT", "ADMIN"}, logical = Logical.OR)
public Result<String> merchantOrAdminMethod() {
    // 需要商家或管理员角色
}
```

### 3. 需要特定权限
```java
@RequiresPermissions("product:create")
public Result<String> createProduct() {
    // 需要创建商品权限
}

@RequiresPermissions(value = {"product:view", "product:create"}, logical = Logical.AND)
public Result<String> viewAndCreate() {
    // 需要同时拥有查看和创建权限
}
```

## 密码加密

本系统使用 **SHA-256 + Salt** 进行密码加密：

- **算法**：SHA-256
- **迭代次数**：1024
- **盐值**：每个用户随机生成

加密工具类：`com.example.demo.util.PasswordUtil`

```java
// 加密密码
String[] encrypted = PasswordUtil.encryptPassword("123456");
String encryptedPassword = encrypted[0];  // 加密后的密码
String salt = encrypted[1];                // 盐值

// 验证密码
boolean isValid = PasswordUtil.verifyPassword("123456", encryptedPassword, salt);
```

## 配置说明

### application.yml

```yaml
# Shiro 配置
shiro:
  enabled: true
  web:
    enabled: true
  session-manager:
    session-validation-interval: 1800000  # 30分钟
    global-session-timeout: 1800000       # 30分钟
  cookie:
    name: SHIROSESSIONID
    max-age: 1800
    http-only: true
```

### ShiroConfig.java

配置过滤器链规则：

```java
// 公开接口，无需认证
filterChainDefinitionMap.put("/api/auth/register/**", "anon");
filterChainDefinitionMap.put("/api/auth/login", "anon");

// 需要认证的接口
filterChainDefinitionMap.put("/api/auth/logout", "authc");

// 需要特定角色的接口
filterChainDefinitionMap.put("/api/merchant/**", "authc,roles[MERCHANT]");
filterChainDefinitionMap.put("/api/buyer/**", "authc,roles[BUYER]");
filterChainDefinitionMap.put("/api/admin/**", "authc,roles[ADMIN]");
```

## 异常处理

系统提供全局异常处理器 `GlobalExceptionHandler`，自动处理 Shiro 相关异常：

| 异常 | HTTP 状态码 | 说明 |
|-----|-----------|------|
| UnauthenticatedException | 401 | 未认证（未登录） |
| UnauthorizedException | 403 | 未授权（权限不足） |
| AuthenticationException | 401 | 认证失败 |
| AuthorizationException | 403 | 授权失败 |

## 注意事项

1. **Session 管理**：默认使用内存存储 Session，生产环境建议使用 Redis 存储
2. **密码强度**：建议在注册时添加密码强度验证
3. **登录限制**：建议添加登录失败次数限制，防止暴力破解
4. **HTTPS**：生产环境建议启用 HTTPS，保护传输安全
5. **跨域**：如果前后端分离，需要配置 CORS 跨域支持

## 问题排查

### 1. 登录成功但无法访问需要认证的接口

- 检查 Cookie 是否正确传递
- 检查 Session 是否过期
- 查看 Shiro 日志

### 2. 权限不足

- 检查用户类型（userType）是否正确
- 检查 UserRealm 中的权限分配逻辑
- 查看授权日志

### 3. 密码验证失败

- 检查密码加密算法配置
- 检查盐值是否正确存储和读取
- 使用 PasswordUtil 单独测试加密和验证
