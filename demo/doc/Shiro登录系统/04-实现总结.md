# Apache Shiro 登录系统实现总结

## 实现概述

已成功实现基于 Apache Shiro 的用户登录认证和权限管理系统。系统采用**双表设计**：
- **用户信息表（t_user）**：存储用户基本信息
- **用户登录表（t_user_login）**：存储登录凭证和用户类型（商家/买家/管理员）

## 完成的功能

### ✅ 核心功能

1. **用户注册**
   - 支持三种类型用户注册：商家、买家、管理员
   - 密码使用 SHA-256 + 盐值加密
   - 自动校验用户名和手机号唯一性

2. **用户登录**
   - 基于 Shiro 的认证机制
   - 自动验证用户状态（禁用/正常）
   - 记录登录时间和 IP

3. **权限管理**
   - 基于角色的访问控制（RBAC）
   - 细粒度的权限控制
   - 支持角色和权限注解

4. **Session 管理**
   - 支持 Session 缓存（EhCache）
   - 可配置的超时时间
   - 自动清理过期 Session

5. **密码安全**
   - SHA-256 哈希算法
   - 1024 次迭代加密
   - 每个用户独立盐值

## 数据库设计

### 表结构

#### t_user（用户信息表）
```sql
- id: 用户ID
- username: 用户名（唯一）
- password_hash: 密码哈希（保留字段）
- nickname: 昵称
- phone: 手机号
- email: 邮箱
- status: 状态（0禁用 1正常 2锁定）
- last_login_time: 最后登录时间
- last_login_ip: 最后登录IP
- created_at: 创建时间
- updated_at: 更新时间
```

#### t_user_login（用户登录表）
```sql
- id: 主键
- user_id: 用户ID（关联t_user）
- username: 登录用户名（唯一）
- password: 加密密码
- user_type: 用户类型（MERCHANT/BUYER/ADMIN）
- salt: 密码盐值
- status: 状态（0禁用 1正常）
- created_at: 创建时间
- updated_at: 更新时间
```

## 权限设计

### 用户类型与权限

| 用户类型 | 角色代码 | 权限列表 |
|---------|---------|---------|
| **商家** | MERCHANT | product:create, product:update, product:delete, product:view<br/>order:view, order:process |
| **买家** | BUYER | product:view<br/>order:create, order:view, order:cancel |
| **管理员** | ADMIN | user:*, product:*, order:*, system:* (所有权限) |

### 权限格式

采用 `资源:操作` 格式：
- `product:create` - 创建商品
- `product:view` - 查看商品
- `order:view` - 查看订单
- `user:*` - 用户模块所有权限

## 核心组件

### 1. UserRealm.java
自定义认证和授权 Realm

**认证逻辑**：
```java
protected AuthenticationInfo doGetAuthenticationInfo(AuthenticationToken token)
```
- 查询用户登录信息
- 验证账号状态
- 返回认证信息供 Shiro 验证密码

**授权逻辑**：
```java
protected AuthorizationInfo doGetAuthorizationInfo(PrincipalCollection principals)
```
- 根据用户类型分配角色
- 根据角色分配权限
- 返回授权信息

### 2. ShiroConfig.java
Shiro 配置类

**配置内容**：
- SecurityManager 配置
- 过滤器链配置
- 密码匹配器配置
- EhCache 缓存配置

**过滤规则示例**：
```java
filterChainDefinitionMap.put("/api/auth/login", "anon");           // 公开
filterChainDefinitionMap.put("/api/auth/logout", "authc");         // 需要认证
filterChainDefinitionMap.put("/api/merchant/**", "authc,roles[MERCHANT]"); // 需要角色
```

### 3. AuthService.java
认证业务服务

**核心方法**：
- `register()` - 用户注册
- `login()` - 用户登录（调用 Shiro）
- `logout()` - 用户登出
- `getCurrentUser()` - 获取当前用户
- `changePassword()` - 修改密码

### 4. AuthController.java
认证控制器

**接口列表**：
- `POST /api/auth/register/{type}` - 注册
- `POST /api/auth/login` - 登录
- `POST /api/auth/logout` - 登出
- `GET /api/auth/info` - 获取用户信息
- `POST /api/auth/change-password` - 修改密码
- `GET /api/auth/test/*` - 权限测试接口

### 5. PasswordUtil.java
密码工具类

**功能**：
- 生成随机盐值
- 加密密码（SHA-256 + Salt + 1024次迭代）
- 验证密码

### 6. GlobalExceptionHandler.java
全局异常处理器

**处理异常**：
- `UnauthenticatedException` → 401 未认证
- `UnauthorizedException` → 403 未授权
- `AuthenticationException` → 401 认证失败
- `AuthorizationException` → 403 授权失败

## 使用示例

### 1. 注解方式权限控制

```java
// 需要登录
@RequiresAuthentication
public Result<String> needLogin() {
    return Result.success("已登录");
}

// 需要商家角色
@RequiresRoles("MERCHANT")
public Result<String> merchantOnly() {
    return Result.success("商家专属");
}

// 需要特定权限
@RequiresPermissions("product:create")
public Result<String> createProduct() {
    return Result.success("可以创建商品");
}
```

### 2. 编程方式权限检查

```java
Subject subject = SecurityUtils.getSubject();

// 检查是否登录
if (subject.isAuthenticated()) {
    // 已登录
}

// 检查角色
if (subject.hasRole("MERCHANT")) {
    // 是商家
}

// 检查权限
if (subject.isPermitted("product:create")) {
    // 有创建商品权限
}
```
