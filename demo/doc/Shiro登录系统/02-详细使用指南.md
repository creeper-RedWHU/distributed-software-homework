# Shiro 登录系统 - 快速启动指南

## 项目概述

本项目实现了基于 Apache Shiro 的用户登录认证和权限管理系统，包含以下特性：

✅ **双表设计**：用户信息表（t_user）+ 用户登录表（t_user_login）
✅ **三种用户类型**：商家（MERCHANT）、买家（BUYER）、管理员（ADMIN）
✅ **自定义 Realm**：实现认证和授权逻辑
✅ **密码加密**：SHA-256 + 盐值（Salt）
✅ **权限控制**：基于角色和权限的访问控制
✅ **Session 管理**：支持 Session 缓存和超时控制

## 快速开始

### 1. 确保环境配置

```bash
# 检查 MySQL 是否运行
mysql -u root -p

# 检查 Redis 是否运行（可选）
redis-cli ping
```

### 2. 配置数据库

编辑 `src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/distributed?...
    username: root
    password: your_password  # 修改为你的 MySQL 密码
```

### 3. 启动项目

```bash
# 使用 Maven
mvn spring-boot:run

# 或使用 IDE 运行 DemoApplication.java
```

项目会自动执行以下操作：
- 创建数据库表（t_user、t_user_login、t_user_role 等）
- 初始化测试数据（3个测试账号）

### 4. 验证启动成功

访问健康检查接口：
```bash
curl http://localhost:8080/ping
```

应该返回：`{"code":200,"message":"success","data":"pong"}`

### 方法 3：使用 curl 命令

#### 1. 商家登录

```bash
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -c cookies.txt \
  -d '{"username":"merchant1","password":"123456"}'
```

**响应示例：**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "userId": 1,
    "username": "merchant1",
    "token": "session-id-xxxxx",
    "expiresIn": 1800,
    "userType": "MERCHANT"
  }
}
```

#### 2. 获取当前用户信息

```bash
curl http://localhost:8080/api/auth/info \
  -b cookies.txt
```

#### 3. 测试权限

```bash
# 测试商家权限（应该成功）
curl http://localhost:8080/api/auth/test/merchant \
  -b cookies.txt

# 测试买家权限（应该失败 - 403）
curl http://localhost:8080/api/auth/test/buyer \
  -b cookies.txt

# 测试创建商品权限（商家有此权限）
curl http://localhost:8080/api/auth/test/create-product \
  -b cookies.txt
```

#### 4. 登出

```bash
curl -X POST http://localhost:8080/api/auth/logout \
  -b cookies.txt
```

## 使用场景示例

### 场景 1：商家管理商品

```bash
# 1. 商家登录
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -c cookies.txt \
  -d '{"username":"merchant1","password":"123456"}'

# 2. 创建商品（假设有此接口）
# 商家有 product:create 权限，可以成功
curl -X POST http://localhost:8080/api/product/create \
  -H "Content-Type: application/json" \
  -b cookies.txt \
  -d '{"name":"商品A","price":100}'
```

### 场景 2：买家下单

```bash
# 1. 买家登录
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -c cookies.txt \
  -d '{"username":"buyer1","password":"123456"}'

# 2. 查看商品（买家有 product:view 权限）
curl http://localhost:8080/api/product/list -b cookies.txt

# 3. 创建订单（买家有 order:create 权限）
curl -X POST http://localhost:8080/api/order/create \
  -H "Content-Type: application/json" \
  -b cookies.txt \
  -d '{"productId":1,"quantity":2}'
```

### 场景 3：管理员管理系统

```bash
# 1. 管理员登录
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -c cookies.txt \
  -d '{"username":"admin1","password":"123456"}'

# 2. 创建新管理员（需要 ADMIN 角色）
curl -X POST http://localhost:8080/api/auth/register/admin \
  -H "Content-Type: application/json" \
  -b cookies.txt \
  -d '{"username":"admin2","password":"123456"}'

# 3. 管理员可以访问所有接口（拥有全部权限）
curl http://localhost:8080/api/admin/users -b cookies.txt
```

## 代码中使用权限控制

### 使用注解

```java
// 需要登录
@RequiresAuthentication
public Result<String> someMethod() { ... }

// 需要商家角色
@RequiresRoles("MERCHANT")
public Result<String> merchantMethod() { ... }

// 需要多个角色之一
@RequiresRoles(value = {"MERCHANT", "ADMIN"}, logical = Logical.OR)
public Result<String> merchantOrAdmin() { ... }

// 需要特定权限
@RequiresPermissions("product:create")
public Result<String> createProduct() { ... }

// 需要多个权限
@RequiresPermissions(value = {"product:view", "product:create"}, logical = Logical.AND)
public Result<String> viewAndCreate() { ... }
```

### 编程式权限检查

```java
import org.apache.shiro.SecurityUtils;
import org.apache.shiro.subject.Subject;

public void someMethod() {
    Subject subject = SecurityUtils.getSubject();

    // 检查是否登录
    if (!subject.isAuthenticated()) {
        throw new UnauthenticatedException("请先登录");
    }

    // 检查角色
    if (subject.hasRole("MERCHANT")) {
        // 商家逻辑
    }

    // 检查权限
    if (subject.isPermitted("product:create")) {
        // 创建商品
    }

    // 获取当前用户名
    String username = (String) subject.getPrincipal();
}
```

## 常见问题

### Q1: 登录成功但访问接口返回 401

**原因：** Cookie 未正确传递

**解决：**
- 使用 curl 时加上 `-b cookies.txt` 参数
- Postman 中确保启用了 Cookie 管理
- 浏览器中检查是否允许第三方 Cookie

### Q2: 权限测试接口返回 403

**原因：** 当前用户没有对应权限

**解决：**
1. 检查当前登录用户类型
2. 查看权限矩阵，确认用户是否有该权限
3. 重新登录正确的账号

### Q3: 密码加密失败

**原因：** 密码加密配置错误

**解决：**
1. 检查 `ShiroConfig` 中的 `HashedCredentialsMatcher` 配置
2. 确认数据库中的盐值字段已正确填充
3. 使用 `PasswordUtil` 工具类测试加密

### Q4: Session 过期太快

**解决：** 修改 `application.yml` 中的 Session 超时配置

```yaml
shiro:
  session-manager:
    global-session-timeout: 3600000  # 60分钟
```
